Caso del PDF

El reto de la empresa tecnologica

La organizacion detecta una baja de productividad por uso inadecuado de Internet en horario laboral. El objetivo es configurar FortiGate-VM para bloquear categorias no relacionadas con el trabajo, como redes sociales o streaming, y reforzar la seguridad del entorno corporativo.

Operaciones HT08

Configuracion explicada paso a paso

1

Seleccionar modo de inspeccion

Para analisis profundo, el PDF propone usar modo proxy.

config system global
set inspection-mode proxy
end
2

Activar NGFW Policy-Based

Este modo permite que las reglas web se basen en identidad, aplicacion y categoria de contenido.

config system settings
set ngfw-mode policy-based
end
3

Aplicar perfil de filtrado web

La politica acepta el trafico permitido, habilita UTM y usa el perfil corporativo de Web Filter.

config firewall policy
edit 1
set name "Filtrado-Web-Corporativo"
set srcintf "lan"
set dstintf "wan"
set action accept
set utm-status enable
set webfilter-profile "corporativo"
next
end

Reflexion final

        
Preguntas con respuestas aplicadas

      

      

        

          

            
            
          

          

            
¿Que funcion cumple el modo de inspeccion?

            
Define como FortiGate analiza el trafico. Flow-based revisa paquetes con menor latencia; Proxy-based reconstruye sesiones y permite inspeccion mas profunda para aplicar filtrado web con mayor detalle.

          

        

        

          

            

          

          

            
¿Que ventajas ofrece la inspeccion SSL?

            
Ayuda a controlar contenido dentro de conexiones HTTPS. Sin inspeccion SSL, parte del trafico cifrado puede pasar sin evaluarse por categoria, reputacion o aplicacion.

          

        

        

          

            

          

          

            
¿Como evitar falsos positivos?

            
Se deben revisar logs, crear listas de excepcion, probar reglas por grupos piloto y documentar cambios. La politica debe bloquear riesgos sin interrumpir sitios necesarios para el trabajo.

          

        

        

          

            

          

          

            
¿Como adaptarlo a educacion o gobierno?

            
En educacion se prioriza aprendizaje seguro, bloqueo de contenido adulto y horarios flexibles. En gobierno se refuerzan trazabilidad, cumplimiento, registros y acceso por roles.

          

        

        

          

            

          

          

            
¿Como mejora futuras redes virtualizadas?

            
La practica enseña a diseñar politicas repetibles, medibles y escalables. En redes virtualizadas, esto facilita migraciones, segmentacion, auditorias y despliegues consistentes en nuevos entornos.
